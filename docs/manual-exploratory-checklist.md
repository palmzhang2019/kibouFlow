# 上线前手动探索测试清单

## 0. 执行说明

- 执行环境：本地或预发，Node `>=20.9.0`，`npm run dev` 或预发地址可访问。
- 测试账号：如无登录系统，本项目按游客主流程验证。
- 记录方式：每项记录“通过/失败 + 现象 + 截图/录屏链接 + 复现步骤”。

---

## 1. 关键页面检查（ZH / JA）

- [ ] 打开 `/zh`，页面可正常渲染，无明显报错。
- [ ] 打开 `/ja`，页面可正常渲染，无明显报错。
- [ ] 顶部导航可见，`GEO` logo 可点击回首页。
- [ ] `/zh/guides`、`/ja/guides` 能进入，至少一篇文章详情可打开。
- [ ] `404` 页面行为正常（访问一个不存在的路径）。

---

## 2. 关键主流程走通

### 2.1 Trial 流程

- [ ] 进入 `/zh/trial`，填写必填字段并提交成功跳转到 `/zh/trial/success`。
- [ ] 进入 `/ja/trial`，填写必填字段并提交成功跳转到 `/ja/trial/success`。
- [ ] 提交中按钮状态变化正常（禁用/loading）。

### 2.2 Partner 流程

- [ ] 进入 `/zh/partner`，填写必填字段并提交成功跳转到 `/zh/partner/success`。
- [ ] 进入 `/ja/partner`，填写必填字段并提交成功跳转到 `/ja/partner/success`。
- [ ] 提交中按钮状态变化正常（禁用/loading）。

---

## 3. 多角色/权限与异常提示检查

- [ ] 缺失必填字段时，前端校验和错误提示符合预期。
- [ ] 快速重复提交触发限流时，提示文案清晰（429 路径）。
- [ ] 后端不可用或 DB 未配置时，错误提示可理解（500/503 路径）。
- [ ] `/api/track` 失败不影响主流程（用户可继续操作）。

---

## 4. 浏览器与设备兼容

### 4.1 浏览器

- [ ] Chrome 最新版：主流程全部通过。
- [ ] Safari 最新版（macOS）：主流程全部通过。
- [ ] Edge（可选）：关键页面与表单验证。

### 4.2 设备与分辨率

- [ ] 桌面（>=1280px）：布局正常，无重叠/截断。
- [ ] 平板（768px 附近）：导航与卡片栅格正常。
- [ ] 手机（390px 附近）：菜单、表单、按钮可用，无遮挡。

---

## 5. 文案与体验检查

- [ ] 中文与日文文案无错字、乱码、截断。
- [ ] 指南页标题、描述、CTA 语义一致。
- [ ] 成功页文案与“下一步”说明准确。
- [ ] 加载/提交反馈及时，无明显卡顿或误导。

---

## 6. Hydration 与扩展干扰复验

- [ ] 普通模式下打开首页，确认无阻断性 hydration 错误。
- [ ] 无痕模式复测（排除浏览器扩展影响）。
- [ ] 如安装 Grammarly 等扩展，确认即使有属性注入也不影响功能。

---

## 7. 发布前最终回归

- [ ] `zh` 与 `ja` 首页、guides、trial、partner 全链路各走一遍。
- [ ] 所有成功页可达，返回首页按钮可用。
- [ ] 无 P0/P1 阻断问题；遗留问题有单号与规避说明。
