# 上线前手动探索测试清单

> 本文档整合了 Hermes Agent 执行指令模板。原始文件 `hermes-manual-test-instruction.md` 已归档。

## 0. 执行说明

### 基础执行说明

- 执行环境：本地或预发，Node `>=20.9.0`，`npm run dev` 或预发地址可访问。
- 测试账号：如无登录系统，本项目按游客主流程验证。
- 记录方式：每项记录”通过/失败 + 现象 + 截图/录屏链接 + 复现步骤”。

### Hermes Agent 补充说明

如由 Hermes 测试代理执行，按以下补充说明执行：

**执行前准备：**
1. 不需要启动项目，不需要执行任何构建或本地运行命令。
2. 使用内网基准地址直接访问：`192.168.1.90`。
3. 首先验证基准地址可达；若不可达，直接标记 `BLOCKED` 并结束执行。
4. 记录测试环境：OS/浏览器版本、分辨率、网络条件、是否开启浏览器扩展。

**执行要求：**
1. 必须按 checklist 的章节顺序执行。
2. 每个检查项都要给出明确结论：
   - `PASS`：满足预期
   - `FAIL`：与预期不符
   - `BLOCKED`：受环境/权限限制无法验证
3. 遇到失败时必须补充：实际现象、复现步骤（最短路径）、预期行为、严重级别（P0/P1/P2/P3）、证据路径。

**通过标准（判定规则）：**
- 所有 P0/P1 问题必须为 0，才可判定”可发布”。
- 若存在 `BLOCKED`，需明确风险与补测计划。
- 若存在 P2/P3，可”有条件可发布”，但必须给出回归计划。

## 1. 关键页面检查（ZH / JA）

- [ ] 打开 `/zh`，页面可正常渲染，无明显报错。
- [ ] 打开 `/ja`，页面可正常渲染，无明显报错。

## 2. 关键主流程走通

### 2.1 Trial 流程

- [ ] 进入 `/zh/trial`，填写必填字段并提交成功跳转到 `/zh/trial/success`。
- [ ] 进入 `/ja/trial`，填写必填字段并提交成功跳转到 `/ja/trial/success`。

- [ ] 进入 `/ja/partner`，填写必填字段并提交成功跳转到 `/ja/partner/success`。
- [ ] 提交中按钮状态变化正常（禁用/loading）。

## 3. 多角色/权限与异常提示检查

- [ ] 缺失必填字段时，前端校验和错误提示符合预期。
- [ ] 快速重复提交触发限流时，提示文案清晰（429 路径）。
- [ ] 后端不可用或 DB 未配置时，错误提示可理解（500/503 路径）。
- [ ] `/api/track` 失败不影响主流程（用户可继续操作）。

## 4. 浏览器与设备兼容

### 4.1 浏览器

- [ ] Chrome 最新版：主流程全部通过。
- [ ] Safari 最新版（macOS）：主流程全部通过。
- [ ] Edge（可选）：关键页面与表单验证。

### 4.2 设备与分辨率

- [ ] 桌面（>=1280px）：布局正常，无重叠/截断。
- [ ] 平板（768px 附近）：导航与卡片栅格正常。
- [ ] 手机（390px 附近）：菜单、表单、按钮可用，无遮挡。
