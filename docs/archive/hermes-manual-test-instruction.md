# Hermes Agent 手动探索测试执行指令

你是测试执行代理。请严格按照下述步骤，对 `kibouFlow` 项目进行手动探索测试并输出结构化结果。

## 0) 目标与范围

- 目标：依据 `docs/manual-exploratory-checklist.md` 完成上线前手动验证。
- 范围：`/zh`、`/ja` 首页、guides、trial、partner、success 页面及关键交互。
- 输出：逐项“通过/失败/阻塞”，并附证据（截图/录屏/日志）。

## 1) 执行前准备

1. 不需要启动项目，不需要执行任何构建或本地运行命令。
2. 使用我提供的内网基准地址直接访问并测试：`192.168.1.90`。
3. 首先验证基准地址可达；若不可达，直接标记 `BLOCKED` 并结束执行。
4. 记录测试环境：
   - OS / 浏览器版本
   - 分辨率
   - 网络条件（正常/弱网）
   - 是否开启浏览器扩展（如 Grammarly）

## 2) 执行要求

1. 必须按 checklist 的章节顺序执行：
   - 关键页面检查
   - 关键主流程（Trial / Partner）
   - 多角色/权限与异常提示
   - 浏览器与设备兼容
   - 文案与体验
   - Hydration 与扩展干扰复验
   - 发布前最终回归
2. 每个检查项都要给出明确结论：
   - `PASS`：满足预期
   - `FAIL`：与预期不符
   - `BLOCKED`：受环境/权限限制无法验证
3. 遇到失败时必须补充：
   - 实际现象
   - 复现步骤（最短路径）
   - 预期行为
   - 严重级别（P0/P1/P2/P3）
   - 证据路径（截图或录屏）

## 3) 关键执行细则

### 3.1 页面可达性

- 验证 `/zh`、`/ja`、`/zh/guides`、`/ja/guides`、任一 guides 详情页、`/zh/trial`、`/ja/trial`、`/zh/partner`、`/ja/partner` 可访问。
- 验证不存在路径返回 404 行为正常。

### 3.2 Trial 与 Partner 流程

- Trial：分别在 `zh/ja` 完成必填项提交，确认可到达 success 页。
- Partner：分别在 `zh/ja` 完成必填项提交，确认可到达 success 页。
- 验证提交中按钮状态（禁用/loading）是否合理。

### 3.3 异常与鲁棒性

- 缺失必填字段时的校验与提示。
- 重复提交触发限流时提示是否清晰。
- DB 未配置或后端异常时，提示是否可理解。
- `/api/track` 异常不应阻断主要流程。

### 3.4 兼容性与体验

- 浏览器：Chrome、Safari（Edge 可选）。
- 分辨率：桌面、平板、手机窄屏。
- 文案：中/日文无错字、乱码、截断。
- 交互：按钮、导航、语言切换、返回首页正常。

### 3.5 Hydration 与扩展干扰

- 普通模式检查是否有阻断性 hydration 错误。
- 无痕模式复测（排除扩展干扰）。
- 开启 Grammarly 等扩展复测，确认功能不中断。
