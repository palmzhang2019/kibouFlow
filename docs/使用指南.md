## 2. 环境与启动

**要求**：Node.js `>=20.9.0`（见 `package.json` 的 `engines`）。

```bash
npm install
npm run dev
```

- 开发服务器默认：`http://localhost:3000`（与 Playwright `baseURL` 使用的 `127.0.0.1:3000` 等价访问即可）。
- 若安装或构建报错，先执行 `node -v` 确认版本。

**生产形态自测**（可选）：

```bash
npm run build
npm run start
```

## 4. 常用命令速查

| 命令 | 说明 |
|------|------|
| `npm run dev` | 开发模式（Next.js，当前脚本带 `--webpack`） |
| `npm run build` / `npm run start` | 生产构建与启动 |
| `npm run lint` | ESLint |
| `npm run test` | 全量 Vitest（`vitest run`） |
| `npm run test:watch` | Vitest 监听 |
| `npm run test:unit` | 仅单元测试 |
| `npm run test:integration` | 集成测试（如 API、`llms`、`robots` 路由） |
| `npm run test:e2e` | Playwright；会启动或复用本机 `:3000` 上的 dev（见 `playwright.config.ts`） |

**建议习惯**：改动了 `src/app`、`components/seo`、`lib` 中与内容/路由相关的逻辑后，至少执行 `npm run test:unit`；涉及页面渲染与 JSON-LD 时再加 `npm run test:e2e`。

## 5. GEO / SEO 能力怎么用（开发者视角）

以下为「代码与路由在哪里」的索引，便于排查与扩展。

| 能力 | 典型位置 | 说明 |
|------|-----------|------|
| 通用 JSON-LD 输出 | `src/components/seo/JsonLd.tsx` | 统一 `application/ld+json` |
| 站点 / 组织 | `WebSiteJsonLd.tsx`、`OrganizationJsonLd.tsx` | 多挂在 `[locale]/layout.tsx` 与首页 |
| 文章 / FAQ / HowTo | `ArticleJsonLd.tsx`、`FAQPageJsonLd.tsx`、`HowToJsonLd.tsx` | 与指南详情、FAQ 页等组合使用 |
| 面包屑 | `BreadcrumbJsonLd.tsx`、`src/lib/seo/breadcrumbs.ts` | 按路由生成 `BreadcrumbList` |
| 正文抽取 | `src/lib/faq-extractor.ts`、`src/lib/howto-extractor.ts` | MDX 字符串 → 结构化字段 |
| 站点地图 | `src/app/sitemap.ts` | 动态 sitemap |
| robots | `src/app/robots.ts` | 动态 robots |
| 面向模型的文本 | `src/app/llms.txt/route.ts`、`llms-full.txt/route.ts` | 供外部抓取约定路径 |

**人工快速查看**：浏览器打开目标页 → 查看网页源代码或 DevTools → 搜索 `application/ld+json`。

## 6. 内容编辑与 frontmatter（与阶段 3 对齐）

内容位于 **`content/zh/`**、**`content/ja/`** 等目录下的 MDX。README 已列出常用 frontmatter；新增文章时建议尽量补齐：

- `category`、`contentType`、`cluster`、`audience`、`ctaType`、`relatedSlugs`

**与结构化数据相关的约定（摘要）**：

- **`contentType: faq`**：正文用 `## 问题标题` + 段落回答；抽取器会排除部分非问答小节标题（见 `faq-extractor` 与单测）。条目过少时可能不输出 FAQPage。
- **`framework` / `cluster` 且需要 HowTo**：正文中需有约定小节（如「推荐阅读顺序」）下的 **`1.` 有序列表**，否则 HowTo 可能无法生成。

详细规则以 `docs/geo-phase-3-plan.md` 与抽取器实现为准。

## 8. 常见问题

**Q：E2E 报错连不上 3000 端口？**  
A：先单独执行 `npm run dev`，确认本机 3000 未被占用；Playwright 配置了 `reuseExistingServer: true`，可复用已有 dev 进程。

**Q：JSON-LD 里的域名不对？**  
A：检查 `.env.local` 中的 `NEXT_PUBLIC_SITE_URL`（无尾斜杠），修改后重启 dev。

**Q：只改了一篇 MDX，要跑全套测试吗？**  
A：至少跑与内容类型相关的单测（如 `content`、`faq-extractor`、`howto-extractor`）；若动了 frontmatter 语义，建议 `npm run test:unit` 全量更稳。
