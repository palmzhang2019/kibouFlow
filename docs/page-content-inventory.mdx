## 附：contentType 判断逻辑说明

| 判断方式 | 说明 |
|----------|------|
| **frontmatter 显式声明** | `contentType: "faq"` / `"framework"` / `"concept"` / `"case"` / `"cluster"` 等，直接读取 |
| **frontmatter 未声明，由 inferContentType() 推断** | category 为 `problems` → `problem`；`paths` → `path`；`boundaries` → `boundary`（fallback）；`cases` → `case` |
| **目录名隐含** | `content/zh/paths/job-prep-cluster-entry.mdx` 虽在 `paths/` 目录下，但 frontmatter 显式写了 `contentType: "cluster"`，以显式值为准 |

**cluster 的推断逻辑**（[src/lib/content.ts:106-124](src/lib/content.ts#L106-L124)）：slug 含 `hope-sorting` / `goal-unclear` / `push-forward` → `direction-sorting`；含 `japanese` / `language` → `japanese-path`；含 `partner` → `partner-needs`；其余 paths/problems → `job-prep`。

## 附：指南索引页的内容分组逻辑

[guides/page.tsx](src/app/[locale]/guides/page.tsx) 中的分组渲染规则：

| 展示区块 | 内容来源 | 过滤条件 |
|----------|----------|----------|
| Clusters | `getArticlesByContentType(locale, "cluster")` | contentType === "cluster" |
| Case Library | 特殊查找：`allArticles.find(a => a.slug === "case-library")` | slug === "case-library"，另从各 cluster 取一条 sample case |
| FAQs | `getArticlesByContentType(locale, "faq")` | contentType === "faq" |
| Frameworks | `getArticlesByContentType(locale, "framework")` | contentType === "framework" |
| Concepts | `getArticlesByContentType(locale, "concept")` | contentType === "concept" |
| Problems / Paths / Cases（按目录分组） | `CATEGORIES.map` + `allArticles.filter` | 排除 cluster / faq / framework / concept 后的剩余 article |
