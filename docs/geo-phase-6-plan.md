# GEO 第六阶段实施计划：答案页与术语页扩展（可检索资产）

> 前提：后台 + 内容标准化 + 作者体系已上线。

---

## 1. 阶段定位

| 项目 | 值 |
|---|---|
| 阶段编号 | GEO Phase 6 |
| 预估工时 | 5-9 人日 |
| 优先级 | P1 |
| 目标 | 建立可复用的“问答资产层”，提升长尾可见性与引用率 |

一句话目标：把分散内容提炼为可直接命中的答案与术语定义页。

---

## 2. 核心任务

| 编号 | 任务 | 说明 |
|---|---|---|
| G6-1 | `/answers` 频道 | 1 问 1 答短页，关联来源文章 |
| G6-2 | `/glossary` 术语页 | 概念定义、同义词、相关问题 |
| G6-3 | schema 扩展 | `QAPage`（谨慎）、`DefinedTerm`、`FAQPage` 组合 |
| G6-4 | 后台支持 | 可配置问题、标准答案、引用来源 |
| G6-5 | 内链策略 | 文章与 answers/glossary 双向链接 |

---

## 3. 内容治理规则

- 每个答案页必须有来源链接（至少 1 条站内来源）。
- 术语页必须包含“定义 + 不适用边界 + 推荐下一步”。
- 不做无来源的 AI 生成答案直出。

---

## 4. 风险与控制

- 风险：薄页过多，质量稀释。
- 控制：设置最小质量阈值（字数、来源数、更新时间）。
- 风险：QAPage 使用不当。
- 控制：仅真实问答场景输出 `QAPage`，其余用 FAQPage/Article。

---

## 5. 验收标准

- [ ] 至少上线一批可索引答案页（例如 20+）。
- [ ] 术语页可被站内导航与 sitemap 收录。
- [ ] 与原文章形成双向内链。
- [ ] 结构化数据在验证工具中无致命错误。
