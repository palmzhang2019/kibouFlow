# GEO 网站阶段 1 实施计划

## 技术选型确认

| 维度 | 选型 |
|------|------|

- **框架**: Next.js 15 (App Router) + TypeScript
- **样式**: Tailwind CSS 4
- **数据库**: Supabase (PostgreSQL) -- 表单数据 + 自建埋点数据统一存储
- **部署**: 自托管 VPS (Docker + standalone output)
- **语言**: 中日双语 (next-intl 路由切换)
- **埋点**: 自建轻量方案，数据写入 Supabase

## A. 基础工程搭建

### A1. 初始化项目

```bash
npx create-next-app@latest . --typescript --tailwind --app --src-dir
```

安装核心依赖：
- `next-intl` -- 国际化
- `@supabase/supabase-js` -- Supabase 客户端
- `zod` -- 表单校验
- `@tailwindcss/typography` -- 文案排版

### A2. 国际化配置 (next-intl)

- 路由结构: `/zh/...` 和 `/ja/...`
- 默认语言: `zh` (可配置)
- `middleware.ts` 处理语言检测与重定向
- `src/messages/zh.json` 和 `src/messages/ja.json` 存放所有文案
- 所有页面文案都通过 `useTranslations()` 获取，不硬编码

### A3. Supabase 数据库设计

三张核心表：

**trial_submissions (试用表单)**
- `id` (uuid, PK)
- `name` (text) -- 姓名/昵称
- `contact` (text) -- 联系方式
- `current_status` (text) -- 当前所在状态 (单选)
- `main_concern` (text) -- 当前最困扰的问题
- `goal` (text) -- 想达成什么目标
- `willing_followup` (boolean) -- 是否愿意后续联系
- `utm_source`, `utm_medium`, `utm_campaign`, `referrer`, `landing_page` (text)
- `source_note` (text) -- 手动补充来源
- `locale` (text) -- 提交时的语言
- `created_at` (timestamptz)

**partner_submissions (合作表单)**
- `id` (uuid, PK)
- `org_name` (text) -- 机构名称
- `contact_person` (text) -- 联系人
- `contact_method` (text) -- 联系方式
- `org_type` (text) -- 机构类型 (单选)
- `cooperation_interest` (text) -- 合作意向说明
- `utm_source`, `utm_medium`, `utm_campaign`, `referrer`, `landing_page` (text)
- `locale` (text)
- `created_at` (timestamptz)

**tracking_events (埋点)**
- `id` (uuid, PK)
- `event_name` (text) -- 事件名 (page_view / cta_click / form_start / form_submit 等)
- `page_path` (text)
- `element_id` (text, nullable)
- `utm_source`, `utm_medium`, `utm_campaign`, `referrer` (text)
- `session_id` (text) -- 前端生成的匿名 session
- `locale` (text)
- `user_agent` (text)
- `created_at` (timestamptz)

## B. 通用组件开发

### B1. Header

- 固定顶部导航，移动端汉堡菜单
- 导航项: 首页 / 免费试用 / 机构合作 / FAQ
- 右侧: 语言切换按钮 (中/日)
- 移动端优先，响应式断点 `md:` 切换

### B2. Footer

- 一句话品牌定位
- 联系方式
- X (Twitter) 链接
- 导航链接: FAQ / 机构合作 / 隐私说明
- 版权信息

### B3. CTAButtons 通用组件

- 主 CTA: "免费试用" / "开始希望整理" (链接到 `/trial`)
- 次 CTA: "机构合作咨询" (链接到 `/partner`)
- 可配置大小、颜色变体
- 自带埋点 (点击时调用 `track()`)

### B4. LanguageSwitcher

- 切换 `zh` <-> `ja`，保持当前路径不变
- 使用 `next-intl` 的 `useRouter` + `usePathname`

## C. 页面开发

### C1. 首页 (`/[locale]/page.tsx`)

按规范要求，从上到下包含以下模块：

1. **HeroSection** -- 一句话定位 + 补充解释 + 主CTA + 次CTA。首屏轻快直接，不堆段落
2. **ProblemSection** -- 4 个典型用户问题卡片 (不知道先补日语还是先求职 / 资料方向乱 / 不知道问谁 / 想先整理清楚)
3. **ValueSection** -- 三段式价值表达 (先整理希望 -> 再判断路径 -> 再导向下一步)
4. **FlowSection** -- 4 步流程图 (提交情况 -> 希望整理 -> 下一步建议 -> 进入试用/合作)
5. **AudienceSection** -- 适合谁 / 不适合谁，双栏对比
6. **底部 CTA 区** -- 重复主CTA + 次CTA

SEO: 独立 `generateMetadata()` 输出 title / description / OG

### C2. 免费试用页 (`/[locale]/trial/page.tsx`)

1. **试用适合谁** -- 4 点说明
2. **试用会得到什么** -- 4 点说明 (希望整理入口 / 初步归类 / 方向建议 / 是否继续推进的判断)
3. **当前不适合什么** -- 边界说明
4. **TrialForm** -- 表单组件 (字段见数据库设计)
   - 使用 `zod` 做前端+后端双重校验
   - UTM 参数自动从 URL/cookie 读取填入隐藏字段
   - 提交走 `/api/trial` API Route
5. **提交后说明** -- 明确下一步预期

**成功页** (`/[locale]/trial/success/page.tsx`): 确认提交成功 + 说明后续流程 + 返回首页CTA

### C3. 机构合作页 (`/[locale]/partner/page.tsx`)

1. **适合合作的机构类型** -- 4 类 (日语学校 / 求职支援 / 外国人支援 / 轻咨询顾问)
2. **你能为他们减少什么** -- 4 项成本削减说明
3. **合作方式** -- 最轻版本 (免费试点 / 小范围转介 / 前置整理合作)
4. **当前合作边界** -- 明确当前是试点阶段
5. **PartnerForm** -- 合作申请表单
   - 语气偏稳重，不像招商加盟

**成功页** (`/[locale]/partner/success/page.tsx`): 类似试用成功页

### C4. FAQ 页 (`/[locale]/faq/page.tsx`)

- 10 个 FAQ 条目，使用手风琴组件
- 每个回答先结论再补充，简短
- 部分答案末尾附 CTA 链接
- FAQ 内容存在 `messages/zh.json` 和 `messages/ja.json` 中，方便维护

## D. API 路由开发

### D1. `POST /api/trial`

- 接收试用表单数据
- `zod` 校验
- 写入 Supabase `trial_submissions` 表
- 简单 rate-limit (同 IP 10 分钟内不超过 3 次)
- 返回成功/失败 JSON

### D2. `POST /api/partner`

- 接收合作表单数据
- `zod` 校验
- 写入 Supabase `partner_submissions` 表
- 简单 rate-limit
- 返回成功/失败 JSON

### D3. `POST /api/track`

- 接收埋点事件
- 写入 Supabase `tracking_events` 表
- 无需严格校验，fire-and-forget 风格
- 使用 `navigator.sendBeacon` 在前端发送以避免阻塞

## E. 埋点与来源追踪

### E1. tracking.ts 工具函数

```typescript
// 核心函数
track(eventName: string, metadata?: Record<string, string>)

// 预定义事件
trackPageView(path: string)
trackCTAClick(ctaId: string, page: string)
trackFormStart(formType: string)
trackFormSubmit(formType: string)
```

- 每次调用自动附带: session_id, utm 参数, locale, user_agent, page_path
- 使用 `sendBeacon` 或 `fetch` keepalive 发送

### E2. utm.ts 来源参数

- 页面加载时从 URL 解析 `utm_source`, `utm_medium`, `utm_campaign`, `ref`
- 存入 `sessionStorage`，后续表单提交和埋点自动带上
- 同时记录 `document.referrer` 和 `landing_page`

### E3. 埋点覆盖 (按规范要求)

- 首页 Hero CTA 点击
- 首页次 CTA 点击
- 试用页表单开始填写 (focus 第一个字段时触发)
- 试用页表单提交成功
- 合作页 CTA 点击
- FAQ 页进入与跳出
- 所有页面的 page_view

## F. SEO / GEO 基础

### F1. 每页独立 Metadata

每个页面使用 `generateMetadata()` 导出独立的:
- `title` -- 如 "GEO | 在日本发展，先把希望整理清楚"
- `description`
- `openGraph` (title, description, image, url, locale)
- `alternates` (中日双语 hreflang)

### F2. 结构化数据

首页添加 `Organization` JSON-LD；FAQ 页添加 `FAQPage` JSON-LD。

### F3. 技术 SEO

- `/public/robots.txt` -- 允许所有爬虫
- `/app/sitemap.ts` -- 动态生成 sitemap.xml (包含所有中日双语页面)
- 语义化 HTML: 合理的 H1/H2 层级
- 使用 SSR 保证首屏正文可抓取

## G. 部署 (自托管 VPS)

### G1. Docker 化

- `Dockerfile`: 基于 Next.js standalone output 的多阶段构建
- `docker-compose.yml`: 包含 Next.js 服务
- Supabase 使用云托管版，不自建

### G2. 配置

- `.env.example` 列出所有环境变量:
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - `SUPABASE_SERVICE_ROLE_KEY`
  - `NEXT_PUBLIC_SITE_URL`

## H. 双语文案结构

`messages/zh.json` 和 `messages/ja.json` 采用统一结构:

```json
{
  "common": { "nav", "cta", "footer" },
  "home": { "hero", "problems", "value", "flow", "audience" },
  "trial": { "title", "forWhom", "benefits", "notFor", "form", "success" },
  "partner": { "title", "types", "benefits", "boundary", "form", "success" },
  "faq": { "title", "items": [...] },
  "metadata": { "home", "trial", "partner", "faq" }
}
```

第一版先完成中文文案，日文版可在中文定稿后翻译填入。
